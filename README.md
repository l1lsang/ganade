# Discord 인증 역할 봇

수동 인증 티켓, 종교 역할 선택, MBTI 토글 역할과 OpenAI 기반 듀 가나디 캐릭터 채팅을 제공하는 Discord 봇입니다. 인증은 사용자가 티켓을 만들고, 관리자 역할이 티켓 안에서 신분증 또는 학생증을 직접 확인한 뒤 승인하는 방식입니다.

## 준비

1. Discord Developer Portal에서 봇을 만들고 `DISCORD_TOKEN`, `DISCORD_CLIENT_ID`를 준비합니다.
2. 서버에 초대할 때 OAuth2 scope는 `bot`, `applications.commands`를 선택합니다.
3. Bot 권한은 최소 `Manage Roles`, `Manage Channels`, `Manage Messages`, `Manage Webhooks`, `Read Message History`, `Ban Members`, `Attach Files`, `Embed Links`, `Mention Everyone`, `Create Expressions` 또는 `Manage Expressions`, `Use External Emojis`가 필요합니다.
4. 환영 메시지와 멤버 정보를 쓰려면 Discord Developer Portal의 Bot 설정에서 `Server Members Intent`를 켭니다.
5. 채팅 글자수 레벨을 집계하려면 같은 화면에서 `Message Content Intent`도 켭니다. 음성방 체류 집계에는 봇 초대 시 `View Channels` 권한이 필요합니다.
6. 초대자 표시를 쓰려면 봇에 `Manage Server` 권한이 있어야 초대 목록을 읽을 수 있습니다.
7. 봇의 최고 역할을 봇이 지급하거나 생성할 역할보다 위로 올려야 합니다.
8. Firebase Console에서 Realtime Database를 만들고 서비스 계정 키를 준비합니다.
9. `.env.example`을 참고해서 `.env`를 만듭니다.

## Firebase 데이터 저장

출석, 경제, 호감도, 생일, 경고, 서버 설정, 익명채팅 추적, 활동 레벨은 모두 Firebase Realtime Database에 저장됩니다. 기본 경로는 `ganadi-bot/stores`이며 `FIREBASE_DATABASE_ROOT`로 바꿀 수 있습니다.

```env
DATA_STORAGE_DRIVER=firebase
FIREBASE_DATABASE_URL=https://PROJECT_ID-default-rtdb.asia-southeast1.firebasedatabase.app
FIREBASE_DATABASE_ROOT=ganadi-bot

# 서비스 계정 JSON 전체를 한 줄로 넣는 방법
FIREBASE_SERVICE_ACCOUNT_JSON={"type":"service_account","project_id":"..."}

# 또는 서비스 계정 값을 나누어 넣는 방법
FIREBASE_PROJECT_ID=
FIREBASE_CLIENT_EMAIL=
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
```

로컬이나 Google Cloud 환경에서 Application Default Credentials를 설정했다면 서비스 계정 환경변수는 생략할 수 있습니다. 서비스 계정 키와 `.env`는 Git에 올리지 마세요. Firebase Admin SDK는 클라이언트용 보안 규칙과 별도로 관리자 권한으로 접근합니다.

기존 `data/*.json` 파일이 있으면 봇이 처음 시작할 때 대응하는 Firebase 저장 경로가 비어 있는 경우에만 자동으로 이전합니다. Firebase에 이미 데이터가 있으면 덮어쓰지 않으며 기존 JSON 파일도 삭제하지 않습니다. `DATA_STORAGE_DRIVER=local`은 테스트나 긴급 점검 용도로만 사용할 수 있습니다.

듀 가나디 캐릭터 채팅에는 `OPENAI_API_KEY`가 필요합니다. 기본 채팅 모델과 호출 제한은 다음 환경변수로 조정할 수 있습니다.

```env
OPENAI_API_KEY=
OPENAI_CHAT_MODEL=gpt-4.1-mini
OPENAI_VISION_MODEL=gpt-4.1-mini
GANADI_CHAT_ENABLED=true
GANADI_CHAT_COOLDOWN_SECONDS=5
GANADI_CHAT_MAX_INPUT_CHARS=1200

# 한국 시간 기준 가나디 안부 3회 + 성경 말씀 하루 1회
BIBLE_MORNING_TIME=08:00
BIBLE_LUNCH_TIME=12:00
BIBLE_EVENING_TIME=19:00
BIBLE_SCHEDULER_GRACE_MINUTES=20
BIBLE_SCHEDULER_INTERVAL_SECONDS=30
```

## 실행

```bash
npm install
npm run sync
npm start
```

`DISCORD_GUILD_ID`를 넣으면 서버 명령어로 빠르게 동기화됩니다. 실행 후 Discord에서 `/업데이트`를 사용하면 현재 서버 명령어를 다시 동기화할 수 있습니다. 전역 명령어가 필요하면 `/업데이트 범위:전역` 또는 `npm run sync:global`을 사용하세요.

서버 안에서 UI 패널을 쓰려면 `/인증패널`, `/문의패널`, `/종교패널`, `/취향역할패널`, `/mbti 설정`을 실행하세요. `/패널`을 실행하면 인증·문의 티켓과 종교 패널을 각각 따로 보냅니다.

## Render 배포

Render Web Service로 배포하면 봇이 `PORT` 환경변수 포트에서 health 서버를 함께 엽니다. Build Command는 `npm install`, Start Command는 `npm start`로 설정하세요. Health Check Path는 `/health`를 사용할 수 있습니다.

Discord 봇만 장시간 실행할 목적이라면 Render Background Worker로 배포하는 방식도 맞습니다. Web Service를 계속 사용할 경우에는 이 health 서버가 Render의 포트 감지 요구사항을 만족합니다.

Discord에서 `애플리케이션이 응답하지 않았어요`가 뜨면 Render 로그에서 아래 줄이 있는지 먼저 확인하세요.

```text
Health server listening on 0.0.0.0:<port>
<봇이름> 로그인 완료
```

두 번째 줄이 없으면 Discord 토큰, 봇 초대, 또는 gateway 연결 문제입니다. Render Free Web Service가 잠들면 Discord gateway도 끊기므로, 계속 켜져 있어야 하는 봇은 Background Worker 또는 잠들지 않는 인스턴스로 운영하는 편이 안정적입니다.

## 웹 설정 UI

Render 또는 로컬에서 열리는 웹사이트 `/`에서 환영 메시지와 자기소개 안내를 설정할 수 있습니다. 공개 URL에 노출되는 관리자 화면이라 `.env` 또는 Render Environment Variables에 `WEB_ADMIN_TOKEN`을 반드시 넣으세요.

웹 UI에서 가능한 설정:

- 봇이 들어간 서버 선택
- 입장 로그 채널 선택
- 퇴장 로그 채널 선택
- 임베드 제목, 추가 메시지, 색상 설정
- 가입자/퇴장자 프로필 이미지 표시
- 가입자 멘션 포함 여부 설정
- 초대자 표시 여부 설정
- 외부 이모지 문자열 입력
- 자기소개 채널, 예시 제목/내용, 하단 문구, 임베드 색상 설정과 미리보기
- 종합·채팅·음성방 활동 랭킹 조회
- 누적 채팅 글자수, 메시지 수, 음성방 체류 시간 요약

외부 이모지는 Discord 형식 그대로 입력하면 됩니다. 단, 봇이 해당 이모지를 사용할 수 있어야 하므로 봇이 이모지가 있는 서버에 있거나, 대상 채널에서 `Use External Emojis` 권한이 있어야 합니다.

```text
<:emoji_name:123456789012345678>
<a:animated_name:123456789012345678>
```

외부 이모지를 서버에 복사하려면 `/이모지추가`를 사용하세요.

```text
/이모지추가 이모지:<:emoji_name:123456789012345678>
/이모지추가 이모지:<a:animated_name:123456789012345678>
/이모지추가 이모지:https://example.com/image.png 이름:my_emoji
/이모지추가 이미지:<첨부파일> 이름:my_emoji
```

이모지 이름은 Discord 제한 때문에 영문, 숫자, 밑줄만 사용할 수 있습니다.

사용 가능한 토큰:

- `{user}`: 가입자 이름
- `{tag}`: 가입자 태그
- `{mention}`: 가입자 멘션
- `{server}`: 서버 이름
- `{memberCount}`: 서버 멤버 수
- `{joinedAt}` / `{joinedRelative}`: 입장 시간
- `{leftAt}` / `{leftRelative}`: 퇴장 시간
- `{createdAt}` / `{createdRelative}`: 계정 생성 시간
- `{inviterMention}` / `{inviterName}` / `{inviterTag}`: 초대자 정보

- `/문의패널 채널:<채널>`: 문의 티켓 생성 전용 패널 게시

- `/설정 인증역할:<역할> 관리자역할:<역할> 로그채널:<채널>`: 인증 역할, 티켓 관리자 역할, 인증 로그 채널 설정

- `/경고 회수 유저:<유저> 개수:<개수> 사유:<사유>`: 유저의 현재 경고 회수
- `/경고 기록 유저:<유저>`: 서버 전체 또는 특정 유저의 경고 전체 기록 파일 확인
- `/경고 설정 자동밴횟수:<횟수>`: 경고 누적 시 영구 밴 기준 설정
- `/경고 로그채널 채널:<채널>`: 경고 담당자·대상자·사유를 기록할 전용 로그 채널 설정
- `/경고 로그해제`: 경고 로그 채널 설정 해제
- `/익명채팅 설정 채널:<채널>`: 해당 채널을 익명채팅방으로 설정
- `/익명채팅 해제`: 익명채팅방 설정 해제
- `/익명채팅 상태`: 현재 익명채팅방 확인
- `/익명채팅 추적 코드:<코드>`: 관리자만 `ㅇㅇ(가상IP)` 작성자의 실제 유저 확인
- `/익명 전달내용:<내용>`: 지정된 익명채팅방에서만 익명 메시지 전달

- `/성경말씀 설정 채널:<채널>`: 매일 세 번 @everyone 안부와 하루 한 번 말씀을 보낼 채널 설정
- `/성경말씀 해제`: 예약 가나디 안부와 말씀 중단
- `/성경말씀 상태`: 현재 채널과 한국 시간 예약 확인
- `/생일 설정 채널:<선택> 축하채널:<선택>`: 생일 등록 UI 채널과 축하 메시지 채널 설정
- `/생일 해제`: 생일 등록과 자동 축하 중단
- `/생일 상태`: 생일 채널과 등록 인원 확인
- `/가나디 호감도 유저:<유저>`: 본인 또는 다른 멤버와 가나디의 호감도 UI 확인
- `/가나디 사진`: `src/ㄱㄴㄷ` 폴더의 가나디 사진 중 한 장을 무작위로 전송
- `/레벨 유저:<유저>`: 본인 또는 다른 유저의 종합 레벨, 채팅, 음성방 활동 확인
- `/랭킹 종류:<종합/채팅/음성방/듀코인>`: 활동 또는 가나디 월드 부자 TOP 10 확인
- `/청소 개수:<개수> 유저:<유저>`: 현재 채널 최근 메시지 삭제
- `/핑`: 봇 응답 상태 확인
- `/종교선택 종교:<선택>`: 기존 종교 역할 지급
- `/종교선택 직접입력:<이름>`: 해당 종교 역할이 없으면 생성 후 지급
- `/업데이트`: 슬래시 명령어 동기화

`/설정`으로 저장한 값은 `.env`의 `VERIFIED_ROLE_ID`, `ADMIN_ROLE_ID`, `LOG_CHANNEL_ID`보다 우선 적용됩니다. 아무 옵션 없이 `/설정`을 실행하면 현재 설정을 확인할 수 있습니다.

종교를 선택하면 `RELIGION_ROLE_PREFIX`로 생성되는 종교 역할 하나만 지급합니다.

`/취향역할패널`은 `NSFW 역할 받기`와 `멘헤라 역할 받기` 버티을 보냅니다. 하나만 누르거나 두 버티을 모두 눌러 두 역할을 함께 받을 수 있고, 이미 가진 역할 버티을 다시 누르면 해제됩니다. `nsfw역할id`와 `멘헤라역할id`에 기존 역할 ID를 입력하면 해당 역할을 사용하고, 비우면 이름이 `NSFW`, `멘헤라`인 역할을 재사용하거나 자동 생성합니다.

## 인증과 문의 티켓

인증 패널의 `인증 티켓 생성` 버튼 또는 `/인증` 명령어를 사용하면 티켓 채널이 생성됩니다. 티켓은 기본적으로 티켓 생성자와 `/설정 관리자역할`로 지정한 역할만 볼 수 있습니다.

티켓 안 버튼:

- `인증 승인`: 관리자 역할만 사용할 수 있으며, 인증 역할을 지급합니다.
- `티켓 닫기`: 티켓 생성자 또는 관리자 역할이 사용할 수 있으며, 채널을 삭제합니다.

`/문의` 또는 문의 패널의 `문의 티켓 생성` 버튼을 사용하면 같은 관리자 역할만 볼 수 있는 비공개 문의 채널이 생성됩니다. 유저는 인증 티켓과 문의 티켓을 하나씩 동시에 열 수 있습니다.

## 출석체크

`/출석체크`를 실행하면 한국 시간 기준으로 하루 1번 출석됩니다. 응답에는 `{n}일째 출석!!` 문구와 누적 출석, 연속 출석, 최고 연속 기록이 표시됩니다.

출석 랭킹은 `/출석체크 작업:출석 랭킹`으로 볼 수 있고, `/출석체크 작업:출석 초기화`는 서버 관리 권한이 있는 사람만 사용할 수 있습니다. 출석 데이터는 Firebase의 `stores/attendance`에 저장됩니다.

## 활동 레벨과 랭킹

일반 채팅은 공백을 제외한 실제 글자수를 누적하고, 음성방은 접속한 체류 시간을 누적합니다. 봇 계정과 Discord 서버에 지정된 AFK 채널 체류는 집계하지 않습니다. 익명채팅의 `/익명` 전달내용도 작성자의 채팅 활동으로 정상 집계됩니다.

음성방에 있는 멤버에게는 `🔊 음성채팅 중` 역할을 자동으로 지급하고 나가면 회수합니다. 역할은 봇 역할 바로 아래로 자동 정렬되어 다른 역할 아이콘에 가려지지 않으며, 서버가 Discord 역할 아이콘(서버 부스트 레벨 2)을 지원하면 일반 채팅 이름 옆에 `🔊` 아이콘이 표시됩니다. Discord의 컴팩트 모드에서는 채팅 옆 역할 아이콘이 표시되지 않습니다.

기본 산정식은 채팅 1글자당 1 XP, 음성방 1분당 10 XP이며, 레벨은 `250 × (레벨 - 1)²` 누적 XP 기준입니다. 다음 환경변수로 운영 서버에 맞게 조정할 수 있습니다.

```env
LEVEL_CHAT_XP_PER_CHARACTER=1
LEVEL_VOICE_XP_PER_MINUTE=10
LEVEL_XP_STEP=250
```

`/레벨`에서 개인 활동을 확인하고 `/랭킹`의 버튼으로 종합·채팅·음성방 랭킹을 전환할 수 있습니다. 웹 설정 UI에도 같은 세 종류의 랭킹과 서버 전체 활동 요약이 표시됩니다. 데이터는 Firebase의 `stores/levels`에 저장되며 음성방 접속 중에는 1분마다 체크포인트를 저장합니다.

## 경고 관리

`/경고 지급`으로 경고를 누적하고, `/경고 설정 자동밴횟수:<횟수>`에 도달하면 대상 유저를 영구 밴합니다. Discord에서 밴은 해제하기 전까지 유지되는 영구 밴입니다.

`/경고 회수`는 현재 경고 수를 줄이고, `/경고 기록`은 서버 전체 기록 또는 특정 유저 기록을 `.txt` 파일로 첨부합니다. 경고 데이터는 Firebase의 `stores/warnings`에 저장됩니다.

`/경고 로그채널 채널:<채널>`을 설정하면 경고 지급과 회수 때마다 처리한 사람, 경고 대상, 사유, 현재 경고 횟수를 전용 임베드로 기록합니다. 로그 채널에서 봇에 `View Channel`, `Send Messages`, `Embed Links` 권한이 필요합니다.

자동 밴이 정상 작동하려면 봇에 `Ban Members` 권한이 있어야 하고, 봇의 최고 역할이 대상 유저의 최고 역할보다 위에 있어야 합니다.

## 익명채팅방

`/익명채팅 설정 채널:<채널>`을 실행하면 해당 채널에서만 `/익명 전달내용:<내용>` 명령어를 사용할 수 있습니다. 채팅에는 작성자가 `ㅇㅇ(10.x.x.x)` 같은 가상 IP 코드로 표시됩니다. 일반 메시지는 익명 처리하지 않습니다.

Discord 봇은 실제 IP 주소를 받을 수 없으므로 `ㅇㅇ(10.x.x.x)` 값은 실제 IP가 아니라 서버 안에서 유저를 구분하기 위한 가상 코드입니다. 서버 관리자 또는 `/설정 관리자역할`로 지정된 역할은 `/익명채팅 추적 코드:<코드>`로 실제 유저와 최근 익명 메시지 링크를 확인할 수 있습니다. 추적 데이터는 Firebase의 `stores/anonymous-chat`에 저장됩니다.

익명채팅방이 정상 작동하려면 봇에 `View Channel`, `Send Messages`, `Manage Webhooks`, `Attach Files`, `Embed Links` 권한이 필요합니다. `/청소` 명령어는 `Manage Messages`와 `Read Message History` 권한이 필요합니다.

## MBTI 역할

`/mbti 설정 채널:<채널>`을 실행하면 I/E, N/S, T/F, J/P 토글 버튼이 있는 패널을 보냅니다. 사용자가 버튼을 누르면 `MBTI | I` 같은 역할을 자동 생성해 지급하고, 같은 축의 반대 역할은 제거합니다.

## 자기소개 반복 안내

`/자기소개 채널:<채널>`을 실행하면 웹 UI에서 설정한 자기소개 예시 임베드가 즉시 게시됩니다. 해당 채널에 멤버가 메시지를 작성할 때마다 이전 봇 안내를 지우고 같은 안내를 맨 아래에 다시 게시합니다. 멤버가 작성한 자기소개 메시지는 삭제하지 않습니다.

웹 UI에서 반복 안내를 끄거나 다시 켤 수 있으며, 제목·예시 내용·하단 문구·색상을 저장하면 임베드도 즉시 교체됩니다. 봇에는 대상 채널의 `View Channel`, `Send Messages`, `Embed Links` 권한이 필요합니다.

## 생일 등록과 자동 축하

서버 관리자가 `/생일 설정`을 실행하면 `🎂ㆍ생일-등록` 채널을 자동으로 만들고 생일 등록 UI를 게시합니다. `채널` 옵션을 선택하면 새 채널 대신 기존 채널을 사용할 수 있고, `축하채널` 옵션으로 실제 생일 축하 메시지를 보낼 채널을 따로 지정할 수 있습니다. 축하 채널을 비우면 등록 채널을 그대로 사용합니다. 자동 생성된 등록 채널은 일반 메시지 작성을 막아 패널이 깔끔하게 보이도록 설정됩니다.

멤버는 `생일 등록/수정` 버튼을 누른 뒤 모달에 `3월 14일` 또는 `03-14`처럼 월·일만 입력합니다. `등록 해제` 버튼으로 언제든 저장된 생일을 지울 수 있으며 생년은 저장하지 않습니다.

한국 시간 오전 9시가 지난 뒤 당일 생일자를 한 번만 실제 태그하고, 가나디 말투의 축하 임베드를 설정한 축하 채널에 전송합니다. 재시작해도 같은 날 중복 축하하지 않도록 전송 날짜를 Firebase의 `stores/birthdays`에 저장합니다.

## 가나디 월드 경제 게임

가나디 월드의 화폐는 `듀코인(DUC) 🦴`입니다. 신규 유저는 500 🦴로 시작하며 듀코인은 현금, 상품권, 니트로, 실제 물건, 암호화폐나 실제 주식과 교환할 수 없는 서버 내부 오락용 가상 재화입니다.

기존 `/출석체크`에 경제 보상을 통합했습니다. 하루 첫 출석에 기본 100~300 🦴, 연속 출석 하루당 +20 🦴, 7일 단위 보너스 500 🦴, 30일 단위 보너스 3,000 🦴와 칭호를 받습니다. 별도의 `/출석` 명령은 만들지 않습니다.

- 지갑: `/지갑`, `/잔액`, `/송금`, `/내정보`, `/부자랭킹`
- 낚시: `/낚시`, `/인벤토리`, `/판매`, `/전체판매`, `/낚시도감`, `/낚시랭킹`
- 장비와 상점: `/상점`, `/구매`, `/장비`, `/강화`, `/아이템정보`, `/칭호`, `/장착`
- 가상 주식: `/주식목록`, `/주식정보`, `/매수`, `/매도`, `/내주식`, `/주식랭킹`, `/시장`
- 미니게임: `/동전던지기`, `/주사위`, `/슬롯`, `/홀짝`, `/블랙멍`, `/미니게임기록`
- 시장 뉴스: `/경제뉴스`, `/오늘의시장`, `/뉴스기록`, `/시장분석`
- 퀘스트: `/퀘스트`, `/일일퀘스트`, `/주간퀘스트`, `/퀘스트완료`, `/퀘스트랭킹`

낚시 기본 쿨타임은 3분이며 장비와 강화 단계에 따라 감소합니다. 강화 실패 시 장비는 파괴되거나 단계가 내려가지 않고 비용만 사용됩니다. 가상 주식 8개 종목의 가격과 경제뉴스는 서버별로 저장되며 현실 투자 정보와 연결되지 않습니다.

미니게임은 1회 최대 500 🦴, 하루 손실 최대 5,000 🦴, 기본 10초 쿨타임을 적용합니다. 10회 이상 연속 플레이하면 휴식을 권하고 모든 결과는 균일한 난수 결과를 바탕으로 처리합니다.

관리자는 먼저 `/경제설정 로그채널:<채널>`을 실행해야 합니다. 이후 `/관리자지급`, `/관리자회수`, `/경제리셋`, `/주가조정`, `/뉴스생성`, `/아이템추가`, `/아이템삭제`, `/유저제재`, `/확률설정`, `/쿨타임설정`을 사용할 수 있으며 모든 조작은 Firebase 내부 감사 로그와 설정 채널 임베드에 함께 남습니다. `/로그조회`로 감사 로그 파일을 받을 수 있습니다.

모든 경제 데이터는 Firebase의 `stores/economy`에 저장됩니다.

## 듀 가나디 캐릭터 채팅

일반 채팅에서 `듀`, `듀야`, `가나디`, `가나디야`처럼 봇을 부르거나 봇을 직접 멘션하면 OpenAI API로 짧은 캐릭터 답변을 생성합니다. `듀얼`, `프로듀서`처럼 이름이 아닌 단어에는 반응하지 않습니다.

봇 및 웹훅 메시지는 무시하며, 기본적으로 같은 사용자의 연속 API 호출에는 5초 쿨다운을 적용합니다. 응답은 채널별로 순서대로 처리되고 Discord 멘션은 생성하지 않습니다.

### 가나디 호감도

모든 멤버는 서버별로 호감도 50에서 시작합니다. 가나디가 답할 때 OpenAI가 사용자의 말에 담긴 의도와 강도를 판단해 호감도를 올리거나 내립니다. 일상적인 말은 작게, 강한 애정이나 심한 폭언은 한 번에도 크게 반영됩니다. 최저치는 -99,999이고 위쪽 상한은 없습니다. API 오류나 쿨다운으로 답변하지 않은 메시지는 호감도에 반영하지 않습니다.

가나디는 현재 호감도와 방금 한 말의 영향을 함께 반영해 답합니다. 호감도가 높으면 더 다정하고 신뢰하는 태도를 보이고, 낮으면 서운해하거나 `(삐짐)`처럼 감정을 표현하고 경계할 수 있습니다.

`/가나디 호감도`를 실행하면 현재 점수, 최근 변화, 진행 바, 관계 단계와 누적 대화 횟수를 임베드 UI로 확인할 수 있습니다. `유저` 옵션을 선택하면 다른 멤버의 호감도도 볼 수 있습니다. 데이터는 Firebase의 `stores/ganadi-affection`에 저장됩니다.

## 가나디 예약 안부와 하루 말씀

`/성경말씀 설정 채널:<채널>`을 실행하면 가나디가 한국 시간 기준 매일 아침 08:00, 점심 12:00, 저녁 19:00에 `@everyone`을 실제로 호출하며 먼저 안부를 묻습니다. 아침에는 잘 잤는지, 점심에는 밥을 챙겼는지, 저녁에는 오늘 하루가 어땠는지를 가나디의 귀여운 말투로 물어보고 응원합니다.

성경 말씀은 하루 세 번 반복하지 않고 그날 처음 보내는 안부에만 한 번 포함합니다. 보통 아침 안부에 포함되며, 아침 전송을 놓쳤다면 점심이나 저녁 중 그날 첫 안부에 포함됩니다. 선택한 성경 구절의 전문을 먼저 출력하고, 그 의미와 시간대에 어울리는 가나디의 따뜻한 덕담을 이어서 보냅니다.

전송 날짜와 하루 말씀 여부를 서버 설정에 저장하므로 재시작해도 같은 시간대 또는 같은 날 말씀을 중복 전송하지 않습니다. 기본 20분 유예 시간 안에 봇이 재시작되면 해당 안부를 보내지만 지난 안부를 몰아서 보내지는 않습니다. OpenAI 호출이 실패하거나 API 키가 없을 때도 미리 준비된 가나디 문구로 전송합니다.

시간은 `.env`의 `BIBLE_MORNING_TIME`, `BIBLE_LUNCH_TIME`, `BIBLE_EVENING_TIME`으로 변경할 수 있습니다. 대상 채널에서 봇에 `View Channel`, `Send Messages`, `Embed Links`, `Mention Everyone` 권한이 모두 있어야 설정할 수 있습니다.

## 개인정보 처리 방향

봇은 신분증/학생증 이미지를 분석하거나 별도 파일로 저장하지 않습니다. 사용자가 티켓 채널에 올린 이미지는 관리자가 직접 확인합니다. 실제 운영 시 서버 규칙에 인증 이미지 처리 동의, 티켓 종료 후 채널 삭제 방침을 함께 안내하는 것을 권장합니다.
